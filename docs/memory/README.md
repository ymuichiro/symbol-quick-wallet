# Memory Logs

This directory stores working memory logs for ongoing development.

## Purpose
- capture decision history
- record implementation intent
- track significant test outcomes
- preserve context for follow-up tasks

## Rules
- append during active work, not just at the end
- use dated files: `YYYY-MM-DD.md`
- keep entries concise and factual
- do not include secrets (private keys, tokens, passwords)

## Suggested Entry Format
- `Time`
- `Decision/Intent`
- `Action`
- `Result`
- `Next`
